# 🚀 ShopSense Deployment Guide (Vercel)

This project is now pre-configured for seamless deployment to **Vercel**. 

## 1. Quick Start

1.  **Commit & Push**: Make sure all these files are committed and pushed to your GitHub repository:
    `https://github.com/pruthvirajtarode/ShopSense-AI-Recommendation-System`
2.  **Vercel Dashboard**: Go to [vercel.com](https://vercel.com) and log in.
3.  **Import Repo**: Click **"New Project"** and select your `ShopSense-AI-Recommendation-System` repository.
4.  **Framework Preset**: Let it be "Other" (Vercel will detect it automatically).
5.  **Build Settings**: Leave defaults.
6.  **Deploy**: Click **Deploy**.

## 2. Project Architecture

To ensure Vercel compatibility, we have structured the project as follows:

*   **Frontend**: A modern, premium landing page in `index.html` (Static).
*   **Backend**: A FastAPI engine in `api/app.py` (Serverless).
*   **Routing**: `vercel.json` ensures all `/api/*` requests go to the Python backend.
*   **Models & Data**: The `models/` and `data/` directories are included in the bundle for the AI engine to access.

## 3. The Streamlit App

The original Streamlit app is still available in `app/streamlit_app.py`. 
> [!NOTE]
> Vercel is optimized for Static sites and Serverless APIs. For long-running interactive Python apps like Streamlit, it is highly recommended to use **Streamlit Cloud** or **Railway.app**.
> However, our new **Premium Landing Page (`index.html`)** provides a faster, Vercel-native alternative that talks to the same AI backend.

## 4. Troubleshooting

*   **Internal Server Error**: Check the Vercel logs. Ensure `requirements.txt` includes everything (it currently has `pandas`, `numpy`, `scikit-learn`, `fastapi`, and `uvicorn`).
*   **Path Errors**: The API uses absolute paths relative to the project root for loading the model. This is already handled in `api/app.py`.
